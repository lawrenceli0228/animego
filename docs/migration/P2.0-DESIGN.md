## 1. 与 Express 的兼容性契约

P2.x 跑 staging 时要跟 Express 做 shape diff,envelope 必须**字节级兼容**。本节是硬契约,P2.x 任何 handler 不允许偏离。

### 1.1 Success envelope

**单资源**(`server/controllers/detail.controller.js:31`):

```json
{ "data": { "anilistId": 12345, "titleChinese": "进击的巨人" } }
```

**列表/分页**(`server/controllers/follow.controller.js:59`):

```json
{ "data": [...], "total": 6493, "page": 1, "hasMore": true, "nextPage": 2 }
```

末页:

```json
{ "data": [...], "total": 6493, "page": 7, "hasMore": false, "nextPage": null }
```

⚠️ **三个关键约束:**
1. 分页字段**顶层平铺**,不在 `meta` 里。
2. `nextPage` 在 `!HasMore` 时**显式 `null`,不能省略字段**。Go 用 `*int` 不带 omitempty,nil → JSON null(encoding/json 标准行为,无需 custom marshaler)。
3. 空列表是 `"data": []` 不是 `"data": null`(Go `[]T{}` 序列化为 `[]`,nil slice 为 `null` —— 必须传 empty slice)。

### 1.2 Error envelope

```json
{ "error": { "code": "NOT_FOUND", "message": "用户不存在" } }
```

HTTP status code 跟 envelope 同步(200/201/400/401/403/404/409/429/500)。

### 1.3 已知的 14 个 error code

| Code | 典型 HTTP | 用途 |
|---|---|---|
| `BAD_REQUEST` | 400 | 参数缺失/格式错(非 validator) |
| `VALIDATION_ERROR` | 400 | go-playground/validator 输出 |
| `INVALID_ACTION` | 400 | 业务规则违反 |
| `INVALID_CREDENTIALS` | 401 | 登录失败 |
| `NO_TOKEN` | 401 | Authorization header 缺 |
| `INVALID_TOKEN` | 401 | JWT 解码失败 |
| `TOKEN_EXPIRED` | 401 | JWT exp < now |
| `UNAUTHORIZED` | 401 | 通用未授权 |
| `FORBIDDEN` | 403 | 角色/权限不足 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `CONFLICT` | 409 | 业务冲突 |
| `DUPLICATE_ERROR` | 409 | 唯一约束违反 |
| `TOO_MANY_REQUESTS` | 429 | rate limit |
| `SERVER_ERROR` | 500 | 兜底未分类 |

`internal/httpx/codes.go` 把 14 个定义成 `const`。P2.x 不允许自创新 code 除非更新本表。

func healthHandler(pool *pgxpool.Pool) http.HandlerFunc {
  return func(w http.ResponseWriter, req *http.Request) {
    ctx, cancel := context.WithTimeout(req.Context(), 2*time.Second)
    defer cancel()
    if err := pool.Ping(ctx); err != nil {
      httpx.Fail(w, httpx.NewError(http.StatusServiceUnavailable, httpx.CodeServerError, "database unreachable", httpx.WithCause(err)))
      return
    }
    httpx.Data(w, http.StatusOK, map[string]any{
      "ok": true, "service": "go-api", "stage": "P2.0", "db": "up",
    })
  }
}

#### Logger middleware(post-review:/health skip)

```go
func RequestLog(logger *slog.Logger) func(next http.Handler) http.Handler {
  return func(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, req *http.Request) {
      // post-review: /health docker probe 每 30s 一次,跳过 log
      if req.URL.Path == "/health" {
        next.ServeHTTP(w, req)
        return
      }
      start := time.Now()
      ww := chi.NewWrapResponseWriter(w, req.ProtoMajor)
      next.ServeHTTP(ww, req)
      // log fields: method, path, status, bytes, duration_ms, request_id, client_ip
      // level: 5xx → Error, 4xx → Warn, else → Info
    })
  }
}
```

#### CORS middleware

```go
import gochicors "github.com/go-chi/cors"

func CORS(clientOrigin string) func(next http.Handler) http.Handler {
  return gochicors.Handler(gochicors.Options{
    AllowedOrigins:   []string{clientOrigin},
    AllowedMethods:   []string{"GET","POST","PUT","DELETE","OPTIONS"},
    AllowedHeaders:   []string{"Authorization","Content-Type"},
    AllowCredentials: true,  // P2.2 cookie auth 需要
    MaxAge:           300,
  })
}
```

#### Recoverer middleware

```go
func Recoverer(logger *slog.Logger) func(next http.Handler) http.Handler {
  return func(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, req *http.Request) {
      defer func() {
        if rec := recover(); rec != nil {
          stack := debug.Stack()
          logger.Error("handler panic", "err", rec, "stack", string(stack), "path", req.URL.Path)
          httpx.Fail(w, httpx.NewError(http.StatusInternalServerError, httpx.CodeServerError, "internal error"))
        }
      }()
      next.ServeHTTP(w, req)
    })
  }
}
```

#### 装配顺序(post-review 锁定)

```
CORS → RequestID → RealIP → RequestLog → Recoverer → Timeout
```

理由(§ 9 决策 #3):
- CORS 最外层 — OPTIONS preflight 直接返,不走完整链
- RequestID 在 CORS 后,业务 endpoint 都有 ID;OPTIONS 没 ID 是 OK 的(机构探测,无需追踪)
- RequestLog 在 RequestID 后(读 ID)
- Recoverer 在 RequestLog 后(panic 时 RequestLog 已 wrap ResponseWriter,能拿到 status)
- Timeout 最里层(只业务 handler 需要)

#### 测试

- `logger_test.go`:200/4xx/5xx level 映射 + /health 不写 log + request_id 透传
- `cors_test.go`:OPTIONS preflight 200 + 正确 header + 非 allow Origin 拒
- `recoverer_test.go`:handler panic → 500 envelope + stack trace 进 log

## 4. 跨 sub-task 决策(post-review 全部已定)

### 4.1 `data` 字段允许 nil 吗?

**禁**。返"操作成功无返回值"用 `Data(w, 200, map[string]any{"ok": true})` 或 `Data(w, 204, struct{}{})`。

### 4.2 sqlc generate 触发?

**手动 + Makefile target**。Pre-commit hook 后置(P2.1 加)。CI:`sqlc vet` 验所有 query。

### 4.3 测试用 testcontainers 还是 pgmock?

**testcontainers**(继承 P1.D)。sqlc 生成 query 直接打 SQL,行为依赖真 Postgres 语义。单测仍用 mock(`Querier` interface,mockgen)。

### 4.4 `Page` 是泛型还是 any?

**泛型**(`Page[T any](w, status, items []T, p)`)。post-review 锁定:跟"explicit > clever"对齐,signature 自文档。

### 4.5 `nextPage` 序列化?

**`*int` 不带 omitempty,nil → JSON null**。post-review 锁定:跟 Express byte 完全对齐,无需 custom marshaler。

### 4.6 CORS 位置?

**最外层**。post-review 锁定:`CORS → RequestID → RealIP → RequestLog → Recoverer → Timeout`。OPTIONS preflight 不走完整链。

### 4.7 `/health` log?

**Skip RequestLog**。post-review 锁定:省 2880 条/天 noise log。

### 4.8 `internal/middleware/` vs `internal/httpmw/`?

**`internal/httpmw/`**。post-review 锁定:避开跟 chi/v5/middleware 冲突,免去全部文件 import alias。

## 8. Worktree parallelization(post-review)

| Step | 模块 | Depends on |
|---|---|---|

| C httpx | `internal/httpx/` | - |

| F docs | `docs/`, `go-api/README.md` | A, C, D, smoke |

**Parallel lanes:**
- Lane I: A → smoke
- Lane II: C → D → smoke

Lane I 和 Lane II 都触 `cmd/server/main.go`。**A 只改 pool init 几行,D 只改 middleware 几行**,merge 时手动 rebase 不冲突。建议串行 merge:先 A 后 D。
