02当前 vs 终态

v2.0 现状

  • 前端Vite + React 18 + JSX
  • 后端Express 4 (89+ ep)
  • 语言JS + JSDoc
  • AuthJWT in localStorage
  • RuntimeNode.js 22
  • 包管理npm + package-lock
  • 实时Express + socket.io 同进程
  • 渲染CSR(SPA)

07回滚链

两层回滚。Bun→Node 5 分钟,Next.js→v2.0 SPA 30 分钟。双 Dockerfile 必须从 M6 第一天起就维护。

Layer 1 · Bun → Node

SLA · 5 min · 双 Dockerfile
# VPS 上手动切
ssh root@45.152.65.208 "
  cd /opt/animego && \
  sed -i 's|Dockerfile$|Dockerfile.node|' docker-compose.yml && \
  docker compose build app && \
  docker compose up -d app
"

Layer 2 · Next.js → v2.0 SPA

SLA · 30 min · 全栈回滚
# 万一 Next.js 整体翻车
ssh root@45.152.65.208 "
  cd /opt/animego && \
  git checkout v2.0.x-stable && \
  docker compose down && \
  docker compose build && \
  docker compose up -d
"

迁移期间 v2.0.x 必须保持稳定 tag 在 main 之外,不能 rebase / force push 掉。

08关键决策

所有架构选择在 2026-05-10 一并锁定,不再反复。后续 phase 实施期遇到坑只填,不重议。

TS 范围strict 进度
全量 .jsx → .tsx + 测试转 + strict: false 起步,strict: true 留迁移后单独 PR(20–40 hr)
Next.js 路径Path A / B / C
Path A: 全栈一步到位。不留双栈、不留半成品
Express 命运89+ endpoint
全迁 Next API routes。M1 期间老 Express 跑兜底,M6 部署后下线
RuntimeNode vs Bun
全切 Bun(dev + prod)。Phase 8 canary 兜底,Dockerfile.node fallback 5min 回 Node
socket.io 跑哪P0 决策
独立 ws-server 微服务(端口 3001)。custom server ⛔ Next standalone,选拆服务保 standalone
bcrypt vs Bun.password登录性能
保留 bcrypt。老 hash 跟 Bun.password 不互通,15% 慢可接受
middleware runtimeedge vs nodejs
nodejs。Bun 不支持 edge runtime,顺便保留 jsonwebtoken
Cloudflare 与 ISRcache 协调
CF 只 cache /_next/static/*(1y),HTML 路由 Bypass,让 Next ISR 自管 60s revalidate
testing 框架Bun runtime 下
保留 vitest(跑在 Bun)。bun test API 跟 vi.mock 不兼容,不换
